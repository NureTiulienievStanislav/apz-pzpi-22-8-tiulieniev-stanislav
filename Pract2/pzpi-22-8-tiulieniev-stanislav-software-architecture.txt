2 ЗАВДАННЯ
Метою роботи є дослідження хмарної платформи Amazon Web Services (AWS), аналіз її архітектури, основних сервісів, моделей організації, принципу Shared Responsibility та особливостей побудови комплексних інформаційних систем на базі AWS.
Завдання полягає у підготовці теоретичної довідки з AWS, розгляді реальних архітектурних рішень, підготовці доповіді та презентації з даної теми.

3 ОПИС ВИКОНАНОЇ РОБОТИ
У рамках практичної роботи було досліджено архітектуру хмарної платформи Amazon Web Services (AWS) — однієї з наймасштабніших і найпопулярніших екосистем для розробки, розміщення та підтримки сучасних програмних рішень. Аналіз охоплює історію розвитку AWS, ключові архітектурні принципи, технічні особливості впровадження та реальні сценарії використання основних сервісів платформи.
Amazon Web Services (AWS) — це комплексна хмарна екосистема, створена у 2006 році компанією Amazon. Спочатку платформа пропонувала лише базові послуги інфраструктури, однак сьогодні AWS містить сотні сервісів для зберігання даних, обчислень, аналітики, штучного інтелекту, безпеки, розробки мобільних і web-додатків. Клієнтами AWS є як стартапи, так і глобальні корпорації та урядові організації. Платформа має найбільшу частку ринку серед провайдерів хмарних сервісів.
Архітектура AWS базується на розподіленій інфраструктурі, що охоплює десятки географічних регіонів та сотні дата-центрів по всьому світу. Кожен регіон містить ізольовані зони доступності, що забезпечує високу надійність, відмовостійкість і можливість георозподіленого резервування даних.
Основні компоненти екосистеми AWS організовані у вигляді мікросервісів:
•	Обчислювальні сервіси: EC2 для запуску віртуальних серверів, Lambda для безсерверних обчислень, ECS/EKS для контейнеризації додатків.
•	Сховища: S3 (об'єктне зберігання), EBS (блочне зберігання), EFS (мережева файлова система).
•	Бази даних: реляційні (RDS), NoSQL (DynamoDB), аналітичні (Redshift).
•	Мережеві сервіси: VPC (ізольовані мережі), Route 53 (DNS), CloudFront (CDN).
Для безпеки та управління доступом використовується гнучка система ролей, політик, шифрування, логування та багаторівнева модель Shared Responsibility — у ній AWS відповідає за захист самої інфраструктури, а користувач — за захист власних додатків, даних та ідентифікації.
Архітектура AWS побудована за принципом event-driven, що дозволяє різним компонентам взаємодіяти через черги подій, повідомлення та тригери (наприклад, через SNS, SQS, EventBridge). Такий підхід зменшує залежності між модулями й забезпечує високу масштабованість. У багатьох рішеннях (особливо у serverless-архітектурі) застосовується патерн "Посередник" (Mediator): компоненти не взаємодіють напряму, а всі події передаються через централізований сервіс маршрутизації подій. Це дозволяє швидко додавати нові сервіси, змінювати бізнес-логіку та ефективно розподіляти навантаження між компонентами.
Розглянуто приклад розгортання комплексної інформаційної системи в AWS:
•	Веб-додаток працює на EC2
•	База даних розміщена у RDS
•	S3 використовується для зберігання файлів
•	CloudFront забезпечує швидку доставку контенту
•	CloudWatch використовується для моніторингу
•	Управління здійснюється через AWS Management Console

4 ВИСНОВКИ
Amazon Web Services (AWS) є однією з найпотужніших, надійних і гнучких платформ для розгортання сучасних інформаційних систем. Правильне використання основних сервісів AWS дозволяє будувати масштабовані, стійкі та безпечні рішення для бізнесу будь-якого рівня.
